
start the application from the project root dir:

$ java -jar ./target/myretail-0.0.1-SNAPSHOT.jar



to test the application with curl commands, 
 the following are some examples -

note that the -v option produces verbose output so the return http status codes are evident
(the -s (silent) option removes the progress meter)

open a new terminal window and enter:

$ curl -vs http://localhost:8080/products/13860425

should produce the response:

*   Trying ::1...
* TCP_NODELAY set
* Connected to localhost (::1) port 8080 (#0)
> GET /products/13860425 HTTP/1.1
> Host: localhost:8080
> User-Agent: curl/7.54.0
> Accept: */*
> 
< HTTP/1.1 200 
< Content-Type: application/json
< Content-Length: 96
< Date: Mon, 17 Feb 2020 02:44:40 GMT
< 
{ [96 bytes data]
* Connection #0 to host localhost left intact
{"id":13860425,"name":"Godzilla (Blu-ray)","current_price":{"value":1.78,"currency_code":"USD"}}

optionally, a redirect can be appended to the commands above to view the responses in files:

  $ curl -vs http://localhost:8080/products/13860425 > test_GET_valid_ID.txt 2>&1
	$ curl -vs http://localhost:8080/products/13860429 > test_GET_notfound.txt 2>&1
	$ curl -vs -H "Content-Type: application/json" -X PUT -d '{"value": 1.78,"currency_code":"USD"}' \
	         http://localhost:8080/products/13860425 > test_PUT_valid_ID.txt 2>&1
